Penjelasan :

Secara bahasa gampangnya routing adalah menyambungkan ip dengan network yang berbeda.

Routing IP adalah proses pengiriman data dari satu host dalam satu network ke host dalam network yang lain melalui suatu router. Agar router dapat mengetahui bagaimana meneruskan paket paket ke alamat yang dituju dengan mengunakan jalur terbaik, router menggunakan peta atau tabel routing.

Topologi Sederhana Untuk Latihan Static Routing

Langkah - Langkah :

  1. Pastikan IP client/pc dan IP router sudah di setting sesuai gambar.
  2. Kemudian coba test ping dari client/pc yang ada di router yang sama (ping di pc1 ke RouterMikrotik-1, ping 192.168.1.2). Jika berhasil ke langkah selanjutnya.
  3. kemudian ketikan perintah : ip route print, di router. Artinya router kita baru bisa berkomunikasi dengan network 192.168.1.0 dan network 192.168.3.0
  4. Kita ketikan perintah : ip route add, kemudian ketikan perintah : ip route print bisa dilihat terdapat route baru pada number 0, untuk dst-address 0.0.0.0/0 dan gateway-nya masih kosong.
  5. Kemudian kita ketikan perintah : ip route edit untuk number : ketikan 0 dan untuk value-name : ketikan dst-address (karena kita akan mengedit dst-address). Maksud dari dst-address ini adalah kita akan memperkenalkan router, agar router bisa terhubung atau bisa berkomunikasi dengan network tersebut. Kemudian tekan ctrl + o, untuk save konfigurasi. Bisa dilihat untuk bagian dst-address sudah berubah menjadi 192.168.2.0/29
  6. Ketikan perintah : ip route edit untuk number : ketikan 0 dan untuk value-name : ketikan gateway (karena kita akan mengedit gateway). Untuk gateway masukan IP dimana RouterMikrotik-1 terhubung dengan RouterMikrotik-2, untuk IP-nya 192.168.3.2 (logikanya RouterMikrotik-1 mau berkomunikasi dengan client di RouterMikrotik-2 lewat jalur mana). Kemudian tekan ctrl + o, untuk save konfigurasi. Bisa dilihat untuk bagian gateway sudah berubah menjadi 192.168.3.2
  7. Lakukan hal yang sama dengan RouterMikrotik-2
